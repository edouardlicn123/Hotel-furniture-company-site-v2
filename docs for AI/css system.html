酒店家具公司网站 CSS 设计方案总结

此文档供AI后续开发使用。我们基于多次迭代调整，现将整个 CSS 架构和主题设计总结为一套完整、可维护、易扩展的方案，旨在提供灵活的多主题支持。

1. 设计思路

  • 核心目标：创建一个变量驱动、模块化的 CSS 系统，支持后台一键切换主题，不影响基础结构。
  • 灵感来源：参考高端家具网站（如 Haworth 的极简专业、HAY 的色彩活力、Ligne Roset 的法国优雅），结合酒店场景摄影主导、大白空间、响应式需求。
  • 关键原则：
    • 变量化一切：所有颜色、背景、阴影、渐变通过变量控制。
    • 多主题支持：每个主题文件只负责“赋值 + 细微强化”，易于扩展。
    • 视觉优化：英雄区叠加层确保文字可读；交互柔和；响应式优先。
    • 酒店家具适配：主题以大色块/渐变为基调，突出产品照片（卡片白色对比）。
    • 问题解决导向：高对比、实色覆盖，避免“玻璃感”或染色问题。
  • 用户体验焦点：主题切换无缝；性能高；可访问性强。

2. 系统架构

文件目录结构

app/static/css/
├── custom.css            ← 入口文件（仅导入 base + variables + 当前主题）
├── base.css              ← 基础结构（布局、组件行为、动画，无颜色硬编码）
└── themes/
    ├── variables.css     ← 变量声明（共享清单，不赋值）
    ├── default.css       ← 默认主题（系统将默认使用这个主题）
    ├── dopamine.css      ← 多巴胺活力主题
    ├── macaron.css       ← 马卡龙甜美主题
    └── wafu-sakura.css   ← 和风粉色主题

各文件职责

文件职责
custom.css入口点，顺序导入（base 先、主题后）
base.css核心骨架：布局、组件、响应式，所有视觉属性用变量
themes/variables.css变量清单，仅声明（30+ 语义变量）
themes/*.css具体主题：赋值变量 + 少量强化（如渐变、圆角）

主题切换机制

  • 后台联动：Settings.theme 字段（e.g., 'default'、'wafu-sakura'）
  • 模板动态加载（base.html）： 
    <link rel="stylesheet" href="{{ url_for('static', filename='css/themes/' ~ (settings.theme or 'default') ~ '.css') }}" id="theme-link">
  • 前端动态（可选）：JS 监听下拉切换 href，无刷新生效

3. 主题示例与设计细节

  • dopamine.css：活力爆棚（彩虹渐变 + 不透明菜单）
  • macaron.css：甜美糖果（多色实块 + 深蓝按钮）
  • wafu-sakura.css：和风粉色（统一粉调 + 柔和深粉菜单/按钮）

通用设计细节：

  • Hero 区：背景图 + 半透叠加，文字纯白 + 多层阴影
  • 菜单/按钮：实色块 + 渐变，hover 放大/阴影增强
  • 卡片：白色突出 + 浅渐变，圆润边角
  • 响应式：clamp() 动态字体/间距

4. 实施指南与最佳实践

  • 性能优化：压缩 CSS、懒加载图片、避免 !important
  • 与 Flask 集成：后台添加 theme 下拉，链接加随机版本号防缓存
  • 添加新主题：复制 existing.css → 修改变量 → 添加后台选项
  • 测试与兼容：主流浏览器、桌面/移动设备、Lighthouse 审计（目标 90+ 分）
  • 潜在扩展：
    • 暗模式支持
    • 高级动画（GSAP）
    • 自定义字体（如 Noto Serif JP 提升和风）
    • A/B 测试主题转化率

此方案已成熟稳定，可直接用于后续开发与维护。
